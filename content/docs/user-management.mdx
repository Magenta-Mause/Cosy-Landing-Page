COSY uses an **invitation-based** system — there is no open registration. Users can only join when an Admin or Owner creates an invite for them.

## User Roles

There are three roles with different privilege levels:

| Role | Description |
|---|---|
| **Owner** | Full system access. Can manage all users (including Admins), assign roles, delete users, and access every server. There is typically one Owner per COSY instance (created during installation). |
| **Admin** | Can manage all servers and users (except other Admins and the Owner). Has no resource quotas — can create unlimited servers. |
| **Quota User** | Standard user with resource limits. Can only access their own servers and servers shared with them via [Access Groups](/docs/features/access-management). |

### Role Comparison

| Capability | Owner | Admin | Quota User |
|---|---|---|---|
| Manage own servers | Yes | Yes | Yes |
| Manage all servers | Yes | Yes | No |
| Manage users | Yes | Yes (not Admins) | No |
| Assign roles | Yes | No | No |
| Delete users | Yes | No | No |
| Resource quotas | None | None | Enforced |

## Inviting Users

1. Navigate to the **User Management** page
2. Click **Create Invite**
3. Optionally pre-set a username, role, and resource limits
4. A one-time invite link/key is generated
5. Share the key with the person you want to invite
6. They visit COSY, enter the invite key, set their password, and their account is created

Invites are **single-use** — once redeemed, the key is invalidated. The Owner can revoke pending invites that haven't been used yet.

## Resource Limits

Quota Users have a maximum amount of **memory** and **CPU** they can allocate across all their servers. When creating or modifying a server, COSY checks that the total resource usage stays within the user's quota.

Admins and the Owner can adjust a user's resource limits at any time from the User Management page.

See [Resource Limits](/docs/configuration/resource-limits) for details on how per-server and per-user limits interact.

## Deleting Users

When a user is deleted:

- Their account is permanently removed
- Their game servers are also deleted along with all associated data
- Any Access Group memberships are revoked

Only the Owner can delete users. Admins cannot delete other Admins or the Owner.
