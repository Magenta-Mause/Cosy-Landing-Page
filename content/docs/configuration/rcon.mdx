RCON (Remote Console) is a protocol that allows sending commands to a game server over a network connection. Many game servers support RCON, including Minecraft and Source engine games.

## What Is RCON?

By default, the COSY [Console](/docs/features/console) sends commands to the game server process via Docker attach (stdin). This works for most servers but has limitations — there is no structured response, and some game servers don't read from stdin reliably.

RCON provides a more robust alternative. It's a network protocol specifically designed for sending commands to game servers and receiving structured responses. When RCON is configured, COSY uses it instead of Docker attach.

## Enabling RCON

1. **Enable RCON in the game server** — This is usually done through environment variables or server configuration files. For example, with the `itzg/minecraft-server` image, RCON is enabled by default.

2. **Expose the RCON port** — Add a [port mapping](/docs/configuration/image-and-ports) for the RCON port so COSY can reach it on the host. For example, map container port `25575` to host port `25575`.

   > **Warning:** Only expose the RCON port to the host — do **not** expose it to the internet. RCON traffic is not encrypted and the password is sent in plain text. COSY connects to the RCON port locally on the host, so there is no need for external access.

3. **Configure RCON in COSY** — In the server's **Settings** view, configure the RCON settings:

| Setting | Description |
|---|---|
| **RCON Port** | The port the game server's RCON listens on (the host-side port from the mapping above) |
| **RCON Password** | The password required to authenticate RCON connections |

4. **Start the server** — RCON settings are locked while the server is running, so make sure to configure them while the server is stopped.

## Using RCON

Once configured, you can send commands directly from the COSY [Console](/docs/features/console) view. Commands are sent through RCON to the game server and the response is displayed in the console output.

## Common RCON Commands

### Minecraft

| Command | Description |
|---|---|
| `list` | Show online players |
| `say <message>` | Broadcast a message |
| `op <player>` | Grant operator status |
| `whitelist add <player>` | Add player to whitelist |
| `stop` | Gracefully stop the server |

### Source Engine (CS2, Garry's Mod, etc.)

| Command | Description |
|---|---|
| `status` | Show server status and connected players |
| `changelevel <map>` | Change the current map |
| `kick <player>` | Kick a player |
